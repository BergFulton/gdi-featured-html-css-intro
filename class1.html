Anatomy of an HTML element

  • Element
    • An individual component of HTML
    • Paragraph, heading, table, list, div, link, image, etc.
  • Tag
    • Marks the beginning & end of an element
    • Opening tag and Closing Tag
    • Tags contain characters that indicate the tag's purpose 
      <tagname>Stuff in the middle</tagname>
    • <p> This is a sample paragraph.</p>
An element is an individual unit or component of HTML. The tag marks the start & end of an element, basically saying "Hey! This thing right here? That's a paragraph!" or "Hey, image is here!"

Tag Breakdown

Here's the breakdown of an HTML tag. I'm going to go ahead and get this out there...more than once today, I will forget to close a tag. It happens! So if you find yourself with something that isn't working, first step is to check for typos, second step is to make sure you've closed your tags.

Anatomy of an HTML element

  • Container Element
    • An element that can contain other elements or content
    • A paragraph (<p>) contains text
  • Stand Alone Element
    • An element that cannot contain anything else
    • <br/>
<img/>
Container elements can, surprise, contain things! Like text in a paragraph. Standalone elements can't contain anything else. Break can only be a break, image is only an image.

Anatomy of an HTML element

  • Attribute
    • Provides additional information about the HTML element
    • Class, ID, language, style, identity, source
    • Placed inside an opening tag, before the right angle bracket.
  • Value
    • Value is the value assigned to a given attribute.
    • Values must be contained inside quotation marks.
    • <div id="copyright">©Girl Develop It logo 2016</div>
<img src="my_picture.jpg" />
<a href="http://girldevelopit.com">Girl Develop It</a>
So elements can have attributes, and those attributes give you additional information about the element, such as its class, identity, source, etc. You put attributes inside the opening tag, but before the right angle "greater than" bracket. And value is what is being assigned to a given attribute. These must live inside of quotation marks. Missing quotation marks is another easy to make mistake!

Doctype

The first thing on an HTML page is the doctype, which tells the browser which version of the markup language the page is using.

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML
4.01 Transitional//EN" "http://
www.w3.org/TR/html4/loose.dtd">

<!DOCTYPE html>

* The doctype is case-insensitive.
DOCtype, doctype, DocType and DoCtYpe are all valid.

HTML Tag

After <doctype>, the page content must be contained between <html> tags.

<!DOCTYPE html>
<html>

</html>
Ok, this looks a bit wild, but fear not. In order for a browser to know that the file you're serving it is indeed HTML and not a plate of spaghetti, you've got to make this declaration and proudly annouce that it's HTML. All you need is the doctype html, not the rest of the noise.

Head and Body Tags

Head: The head contains the title of the page & meta information about the page. Meta information is not visible to the user, but has many purposes, like providing information to search engines.

Body: The body contains the actual content of the page. Everything that is contained in the body is visible to the user.

OK, now that we've told the server that this file is HTML, we're done right? But what's this html tag noise? OK, so, there are occasions where you may have other types of code on a page, such as JavaScript, and those get their own tags, such as "script". When we use the HTML tags, we're saying that everything bound in between them is HTML. They're like the big wrapper over our whole work today.

Head and Body Tags: Example

Wait, what? Head and body? So there's a bit of magic that happens between the browser and your HTML. The header is part of that magic. The header tells the browser a lot of valuable information, such as the page's title, and also provides data to search engines. Links to other files also live in the header. But here's the thing. No one will ever know it's there. It just hums away, doing its thing. The body is where the public stuff goes. Anything you put inside the body is visible to the user. This is where your paragraphs, your imgages, everything, goes.

Head and Body Tags

<!DOCTYPE html>
<html>
  <head>
    <title>Title of the page </title>
  </head>
  <body>
    The page content here.
  </body>
</html>

All elements "nest" inside one another

Nesting is what happens when you put other containing tags inside other containing tags. For example, you would put the <p> inside of the <body> tags. The <p> is now nested inside the <body>

Nesting is when you put containing tags inside other containing tags. For example putting paragraphs inside the body. Nesting helps group elements together, and can help you when you're deciding how they should look or behave.

Nesting Order

Whichever element OPENS first CLOSES last

Girl Develop It

The <a> tag surrounds text or images to turn them into links

Ok, now that we've got content, we've got some structure, let's talk about going places with links. Links are a little tricky because they need a few things. They need the "a" or "anchor" tag to kick it off. They also need a HREF, or the actual meat of the link, and they need a title attribute. The A tag surrounds text or images to turn them in to links, but the href tells you where its going. Using Title is also good for people who navigate the internet with screen readers or other assistive technologies.

Link Attributes

Links can have attributes that tell the link to do different actions like open in a new tab, or launch your e-mail program.

<a href="home.html" target="_blank">Link Text</a>

Link opens in a new window/tab with target="_blank"

<a href="mailto:sponsorships@girldevelopit.com">E-mail us!</a>

Adding mailto: directly before the email address means the link will open in the default email program.

Links can also do things like launch an email application, or open themselves in a new tab. Using target="_blank" will open a link in a new tag. putting "mailto:" before an email address will automatically pop open the default email program. So, you see the text of "Email us!" after the close of the mailto attribute? That's what will appear as a link text, generally it'll have an underline. That's where you can get creative and make a user's experience of navigating your content easier. Those linkage words give people an idea of what that link is going to do and where it is going to take them.

Relative vs. Absolute paths for links & images

  • Relative
    Relative paths change depending upon the page the link is on.
    • Links within the same directory need no path information. "filename.jpg"
    • Subdirectories are listed without preceding slashes. "img/filename.jpg"
  • Absolute
    • Absolute paths refer to a specific location of a file, including the domain. "http://www.girldevelopit.com/chapters/detroit"
    • Typically used when pointing to a link that is not within your own domain.
There are two types of links, relative, and absolute. This is a bit of a tough concept to explain, so i'm going to share some links to some explainers in case you'd like a different explainer. So the basics: relative links are relative to the page they appear on. Relative links usually are linking to another page or file within your site. So for instance, remember our file structure? Both the folder "images" and the "index.html" file are in the same overarching folder of "GDI HTML" If you had a file within that images folder that you wanted to link to on your site, you'd need to link to it at img/picture.jpg. If they were at the same level, say you had no images folder, you'd just link to "picture.jpg". Absolute paths refer to a specific file or site, including its domain. TYpically this is if you wanted to link elsewhere, like to your facebook page, or your twitter account.

Alright, we've gotten links going, now let's talk about pictures! To add an image to your page, you use the tag img. IMG is self closing, our first tag like that so far. It has an attribute called "src" or source. Basically, the image tag is asking, "cool, where's your picture located? What's it's source. Now you can link to an image somewhere on the internet, like in this example, or you can use a relative link to something that's already in your images folder, where this would look like "img/logo.jpg". Images also have an attribute caled "alt", also refered to as "alt text". Again this is very important to users who access your page with screen readers or other assistive devices, as those devices will see that alt text and announce it to the user. So someone who is blind would hear a description "Girl Develop It logo", as long as we apply alt text to it. Be careful not to be too verbose in your alt text, but you want to also be sufficiently descriptive. It's a fine balance.

Element: Unordered and ordered lists

<ul>
<li>List Item</li>
<li>AnotherList Item</li>
</ul>

<ol>
<li>List Item</li>
<li>AnotherList Item</li>
</ol>

Unordered list (bullets)

  • List Item
  • Another List Item

Ordered list (sequence)

  1. List Item
  2. Another List Item
Lists come in two flavors, unordered and ordered. Unordered show up as bullets by default (this can be changed), and ordered will display a 1, 2, 3 etc by defualt. You don't have to add the numbers, HTML does that for you, so put them in the sequence you'd like them to be in. To make an unordered list you use the tag "ul". Each item within that list is a "li" for list item. Ordered lists use "ol" for ordered list, and also have "li" inside them. Just as a heads up, li's cannot exist outside of a UL or an OL, they're dependent upon being one of those two things.

Comments

You can add comments to your code that will not be seen by the browser, but only visible when viewing the code.

<!-- Comment goes here -->

Comments can be used to organize your code into sections so you (or someone else) can easily understand your code. It can also be used to 'comment out' large chunks of code to hide it from the browser.

<!-- Beginning of header -->
<div id="header">Header Content </div>
<!-- End of header -->

<!--

Comments are super awesome! Particularly if you are working with someone else, it's helpful to comment your code so that people can follow. I also use this as a way to leave notes for myself to come back to stuff. Comments won't be displayed on the page, but if you view source, comments will be visible, so keep that in mind.
