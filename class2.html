CSS: What is it?

CSS = Cascading Style Sheets

CSS is a "style sheet language" that lets you style the elements on your page.

CSS is works in conjunction with HTML, but is not HTML itself.

And now we arrive at CSS. CSS gives us style!

CSS: What can it do?

All color, text, position, and size

So what can CSS do? A lot actually! CSS controls color, text, position (where and how things appear) and size. CSS can also do some pretty amazing things with animation, but that's a topic for another class. Let's just CSS is pretty darn powerful.

CSS: What does it look like?

The CSS Rule

The CSS Rule

selector {
  property: value;
}
  • A block of CSS code is a rule.
  • The rule starts with a selector.
  • It has sets of properties and values.
  • A property-value pair is a declaration.

Connecting CSS to HTML

3 ways

"Inline"

"Embedded"

"External"

Connecting CSS to HTML: Inline

<p style="color:red">Some text.</p>

Uses the HTML attribute style.

Difficult to use in large projects

Not preferred.

Inlineing your CSS works by adding the HTML attribute "style" and then assigning it a property and value, in this intance, we're saying that we want the text to have the color red. This gets really unweidly, and it can be hard to keep track of all of these rules on a larger project. This method isn't preferred.

Connecting CSS to HTML: Embedded

<head>
  <style type="text/css">
    p {
      color: blue;
      font-size: 12px;
    }
  </style>
</head>

Inside <head> element.

Uses <style> tag.

Can only be used in one html file

Useful if you write HTML email newsletters!

Another way to get HTML & CSS associated with each other is by embedding styles in the head element. Within the head, you can add the style tag, and add styles in there. The downside is that, if you do it this way, that css can only be used in that singular file. If you had another page called puppies.html, you'd either have to copy &past this CSS over there, or start from scratch. It's not terribly useful if you have a multipage site. HOWEVER! Embedded CSS is the bread and butter of email newsletters. For instance, here's an HTML newsletter that I write every week that uses Embedded CSS, but it is only one page, so it's a perfect usecase.

Connecting CSS to HTML: Linked

<head>
  <link rel="stylesheet" type="text/css" href="style.css">
</head>
  • Shared resource for several pages.
  • Reduced file size & bandwidth
  • Easy to maintain in larger projects.
  • Preferred! ðŸ’¯
So if the first two methods are suboptimal for most cases, what do we do? We link! By linking to the CSS file, we enable pages to use that CSS on their page, meaning we don't have to repeat, retype, or remember that CSS. It also reduces your filesize, and keeps all of your style rules tucked in to a nice neat place. So all you do is add the link text in the header like so.

CSS Syntax

Declarations: Property and value of style you plan to use on HTML element.

Declarations end with a semicolon

Declaration groups are surrounded by curly brackets.

selector {
  property: value;
  property: value;
  property: value;
}
CSS Syntax is important, so let's take a look. The CSS sytax begins with the selector, or the thing you are selecting to be styled. Then we add curly braces, we then say what property we'd like to style, then a colon, and then what we want that style to be.

Selector: Element

p {
  property: value;
}

Selects all paragraph elements.

img {
  property: value;
}

Selects all image elements.

CSS Color Values

Your browser can accept colors in many different ways:

  • Color name (ex. red)
  • Hexadecimal value (ex. #FF0000)
  • RGB value (ex. rgb(255, 0, 0))

The 17 standard colors are: aqua, black, blue, fuchsia, gray, grey, green, lime, maroon, navy, olive, purple, red, silver, teal, white, and yellow.

Color is so important in the web, not only because, hey, we like things to look nice, but because it can convey meaning, and help users navigate a site. If you're in branding, colors and color matching are really important to an effective brand strategy. There's a few ways to tell the browser what color you want something to be. There's Color name, so just sayin "red, white, orange", there's a Hexadecimal value, which is a six character code that corresponds to the various aspects of color in it, there is also RGB values, for red green blue. Property: Color

The color property changes the color of the text.

p {
  color: red;
  color: #ff0000;
  color: rgb(255, 0, 0);
}
I have to say, this was one of those things I forgot all the time when I started. I always wanted to say "text color" or "font color" but nope, it's just color! the color property makes your text colorful.

Property: Background-color

The background-color property changes the color of the background.

p {
  background-color: black;
  background-color: #000000;
  background-color: rgb(0,0,0);
}
Background color does what it says on the tin! It sets the background color for the element.

Property Values

Each property can have one or more comma separated values.

p{
  color: white;
  background-color: red;
  font-family: Arial, sans-serif;
}
Sometimes, you'll want to have more than one value asigned to a property. When this happens, just separate them with a comma. So here you see that font-family is set to Arial, and then a sans-serif font is set as the fallback, in case Arial wasn't available. We're going to talk about font-family next

Property: Font-family

The font-family property defines which font is used.

p {
  font-family: "Times New Roman";
  font-family: serif;
  font-family: "Arial", sans-serif;
}

Specific font name

Generic name

Comma-separated list

Font familes decide what fonts get used, and they're assigned in a comma separated list. Offering a fall back font gives your fonts a chance to fail. With something like Arial, it's a pretty safe bet, but if you get in to some fancier web fonts that may not display perfectly on every browser or every machine, it's important to offer a fallback so that users see _something_.

Property: Font-size

The font-size property specifies the size of the font.

p {
  font-size: 12px;
  font-size: 1.5em;
}

Pixels

"em"

Percentage

Anyone want to guess at what font-size does? Yep, tells you how big the font should be. There's a few ways to do this. You can set it in pixels, which is pretty frequently used. You can also set it as "em"s. I'm just going to read the W3C's definition of ems, because they're smart folks and they have a good one: To allow users to resize the text (in the browser menu), many developers use em instead of pixels. The em size unit is recommended by the W3C. 1em is equal to the current font size. The default text size in browsers is 16px. So, the default size of 1em is 16px.

Property: Fonts (shorthand)

p {
  font-style: italic;
  font-weight: bold;
  font-size: 10px;
  font-family: sans-serif;
}
OR 
p {
  font: italic bold 10px sans-serif;
}
You can also save yourself some space by using shorthand properties. For example, you can set font style, font weight, font size and font family all on one line.

Reusing code

As a general coding principle, Don't Repeat Yourself.

To reuse CSS, we use IDs and classes.

Photo credit: Yo Mostro cc

IDs vs. Classes

ID -- Should only apply to one element on a webpage, i.e., a webpage only has one footer.
The "#" is how you tell CSS "this is an id."

Class -- Lots of elements can have the same class, i.e., There can be many warnings on one webpage.
The "." is how you tell CSS "this is a class name."

Selector: ID

#footer {
  property: value;
}

Selects all elements with an id of "footer".

<p id="footer">Copyright 2011</p>

The associated HTML.

Selector: Class

.warning {
  color: red;
}

Selects all elements with a class of "warning".

<p class="warning">Run away!</p>

Cascading priority

Your browser assigns different priorities to CSS depending on the type of selector.

  1. Important! - Most Important
  2. In line CSS
  3. ID
  4. Class
  5. Element - Least Important

Cascading priority

Your browser also assigns priority based on the specificity of the selection. More specific selectors have higher priority.

.main .sale .clearance p{ //Most specific
  color: red;
}
.header .title p{
  color: green;
}
.footer p{ //Least specific
  color: blue;
}

Cascading priority

The tie-breaker is position. Rules lower in the file overwrite rules higher in the file

a{
  background-color: yellow;
}
a{
  background-color: teal;
}
a{ //This rule wins
  background-color: black;
}
